# Etapa 1: Visão e Objetivos

> Perguntas para capturar a visão geral do projeto

**Tempo Estimado:** 5-10 minutos

---

## Sequência de Perguntas

### 1.1 Nome do Projeto

```markdown
Pergunta:
"Qual é o nome do seu projeto?

(Pode ser um nome provisório, depois você pode mudar)"

Exemplos:
- Sistema de Agendamento
- App de Delivery
- Plataforma de Cursos

Captura: visao.nome_projeto
```

### 1.2 Problema

```markdown
Pergunta:
"Qual problema esse projeto resolve?

Me conta como as coisas funcionam HOJE (sem o sistema)
e o que é ruim nisso."

Exemplos:
- "Hoje meus clientes ligam para agendar e eu perco 3 horas por dia no telefone"
- "Minha equipe usa planilhas para controlar estoque e sempre tem erro"
- "Não consigo saber quantos clientes tenho e quanto cada um comprou"

Captura: visao.problema
```

### 1.3 Solução

```markdown
Pergunta:
"Como você imagina que o sistema vai resolver esse problema?

Descreve como seria o cenário IDEAL com o sistema funcionando."

Exemplos:
- "Clientes agendam pelo celular e eu só vejo a agenda pronta"
- "Sistema atualiza estoque automaticamente a cada venda"
- "Dashboard mostra todos os clientes e vendas em tempo real"

Captura: visao.solucao
```

### 1.4 Diferencial

```markdown
Pergunta:
"O que vai fazer seu sistema ser diferente/melhor que outras soluções?

(Se não souber, posso pular essa)"

Exemplos:
- "Vai ser mais simples que os concorrentes"
- "Vai ter integração com WhatsApp"
- "Vai ser específico para [meu segmento]"

Captura: visao.diferencial
Obrigatório: Não
```

### 1.5 Objetivo Principal

```markdown
Pergunta:
"Se o sistema pudesse fazer UMA coisa muito bem, qual seria?

Esse é o objetivo principal."

Exemplos:
- "Reduzir o tempo que gasto atendendo telefone"
- "Nunca mais perder uma venda por falta de estoque"
- "Saber exatamente quanto estou faturando por dia"

Captura: visao.objetivo_principal
```

### 1.6 Objetivos Secundários

```markdown
Pergunta:
"Além do objetivo principal, o que mais você gostaria que o sistema fizesse?

(Liste 2-4 objetivos extras)"

Exemplos:
- "Enviar lembretes automáticos para clientes"
- "Gerar relatórios mensais"
- "Permitir que funcionários acessem"

Captura: visao.objetivos_secundarios
```

### 1.7 Métricas de Sucesso

```markdown
Pergunta:
"Como você vai saber se o sistema está funcionando bem?

Me dá um exemplo de métrica que você gostaria de melhorar."

Template:
- Hoje: [como está]
- Meta: [como quer que fique]
- Prazo: [em quanto tempo]

Exemplos:
- "Hoje atendo 30 ligações/dia, quero reduzir para 5 em 3 meses"
- "Hoje perco 10 vendas/mês por estoque errado, quero zerar em 2 meses"
- "Hoje não sei meu faturamento, quero ver em tempo real"

Captura: visao.criterios_sucesso
```

### 1.8 Timeline

```markdown
Pergunta:
"Você tem uma ideia de prazo? Quando precisa do sistema funcionando?

Vou dividir em fases:
- MVP (versão básica funcionando)
- V1.1 (melhorias)
- V2.0 (versão completa)

(Se não souber, posso sugerir)"

Exemplos:
- MVP: 1 mês
- V1.1: +2 meses
- V2.0: +3 meses

Captura: visao.timeline
```

---

## Resumo da Etapa

Ao finalizar, apresente:

```markdown
"Perfeito! Vamos recapitular a visão do seu projeto:

**Projeto:** [nome]

**Problema:** [descrição resumida]

**Solução:** [descrição resumida]

**Objetivo Principal:** [objetivo]

**Métricas:**
- [métrica 1]
- [métrica 2]

**Timeline:**
- MVP: [prazo]
- Melhorias: [prazo]

Está tudo certo? Posso seguir para a próxima etapa (Personas)?"
```
